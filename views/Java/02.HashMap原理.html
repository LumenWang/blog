    02.HashMap原理

    lumen 2022-12-08 20:53:26 Java

    # 1.HashMap简介

    1. HashMap的实现原理：HashMap由数组+链表(+红黑树 1.8)来实现对数据的存储，方便继承了数组的线性查找和链表的寻址修改
    2. HashMap的存储内容：它存储的内容是键值对(key-value)映射
    3. HashMap的线程安全性：线程不安全，非synchronized，但是在单线程环境下效率很高
    4. HashMap可以接受null键和值，而Hashtable则不能（原因就是equlas()方法需要对象）

    # 2.HashMap的特性

    1. HashMap存储键值对，实现快速存取数据
    2. 允许null键/值
    3. 非同步，线程不安全
    4. 不保证有序(比如插入的顺序)
    5. 实现map接口

    # 3.HashMap底层实现为什么是数组 + 链表？

    1. 首先，读写速度最快的是数组，知道索引的时候，读写只需要O(1)的时间复杂度，而插入、删除操作需要移动数组元素，时间复杂度为O(n)
    2. 我们知道在hashmap中访问元素需要key值，而数组下标是通过 (KEY.hashCode()%数组长度) 得到的，这种情况下会造成哈希碰撞(不同的数据地址计算值相等，原有地址的内容被替换)。因此，当出现哈希碰撞得到了许多地址相同的元素时，将这些下标相同的KEY元素以链表串起来
    3. 链表是从头部插入，不是尾部插入。头插的时间复杂度为O(1)，尾插的时间复杂度为O(n)，这个很好理解
    4. 链表的查找操作需要遍历链表，时间复杂度为O(n)。当链表过长会导致性能降低，所以java在1.8之后引入了红黑树
    5. 关于红黑树的原理可能还需要专门研究，这里知道红黑树接近于平衡二叉树，支持查找/插入/删除/，平均时间复杂度最坏为O(logn)
    6. 红黑树的6个性质：
      • 每个节点非黑即红
      • 根节点是黑的
      • 叶子节点都是黑的
      • 如果一个节点是红的，那么它的两个子节点都是黑的
      • 对于任何节点而言，其到叶子节点的每条路径上都包含相同数目的黑色节点
      • 二叉搜索树的基本性质 任何左节点都<=根节点、任何右节点都>根节点

    # 4.HashMap的主要参数有哪些？

    1. capacity：当前容量，即为数组长度，初始值为1<<4=1*2^4=16，可以扩容，每次扩容都乘以二倍
    2. loadFactor：负载因子，默认为0.75
    3. threshold：扩容阈值，其值为当前capacity * loadFactor
    4. max_capacity：HashMap的极限容量，超过该值将不再扩容，可能产生哈希碰撞
    5. transient Entry<K,V>[ ] table = (Entry<K,V>[]) EMPTY_TABLE;（默认为空{}）核心数据结构，即为数组+链表部分

    数组就是这个Entry类型的table，每个table的元素都是一个Entry

    # 5.HashMap的数据结构是什么样子？如何自己实现一个hashMap

    1. 核心数据结构就是数组+链表
    2. 就是围绕hashmap的数组+链表的数据结构进行设计。在hashmap中的主要表现形式为一个数组，即为Entry<K,V>[] table。而Entry是hashmap的一个内部类

    # 6.hash的计算规则？

    1. 每一个key本身都有一个hashcode，这个hash方法可以对这个hashcode进行重新计算，目的是更少地减少碰撞次数
    2. 除了源码中的hash方法，还可以是数字分析法、平方取中法、分段叠加法、除留余数法、伪随机数法
    3. 抛开HashMap，hash冲突有那些解决办法？开放定址法、链地址法、再哈希法
    static final int hash(Object key) { //扰动函数 最终的目的是让散列分布地更加均匀
  int h;
  return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16); //hashCode()的高16位与低16位进行异或操作
}

    # 7.说说HashMap的存取过程？

    # 7.1 存(put)过程

    1. 存储对象时，我们将K/V传给put方法时，它调用hashCode计算hash从而得到bucket位置，进一步存储，HashMap会根据当前bucket的占用情况自动调整容量(超过Load Facotr则resize为原来的2倍)
    2. 或者另外一种回答：
      • 对key的hashCode做hash操作，然后再计算在bucket中的index
      • 如果没碰撞直接放到bucket里
      • 如果碰撞了，以链表的形式存在buckets后
      • 如果节点已经存在就替换old value(保证key的唯一性)
      • 如果bucket满了(超过阈值，阈值=loadfactor*current capacity，load factor默认0.75)，就要resize扩容

* hashmap.put 源代码
* 往hashmap中放数据
*/
public V put(K key, V value) {
        if (table == EMPTY_TABLE) {
            inflateTable(threshold);//判断如果为空table，先对table进行构造
            //构造通过前面的几个参数
        }
        //首先判断key是否为null，为null也可以存
        //这里需要记住，null的key一定放在table的0号位置
        if (key == null)
            return putForNullKey(value);
        //算出key的hash值，这里调用了前面提到的扰动函数!!!
        int hash = hash(key);
        //根据hash值算出在table中的位置，是数组的下标应该！！没错！！
        int i = indexFor(hash, table.length); //找到i位置对应的Entry
        //放入K\V，遍历链表,如果位置上存在相同key，进行替换value为新的，且将替换的旧的value返回
        for (Entry<K,V> e = table[i]; e != null; e = e.next) {  //这个位置上是一个Entry链表，需要去找到那个k-v对
            Object k;
            if (e.hash == hash && ((k = e.key) == key || key.equals(k))) {
                V oldValue = e.value;
                e.value = value;
                e.recordAccess(this);
                return oldValue;
            }
        }
        modCount++;
        //增加一个entry，有两种情况
        //1、如果此位置存在entry，将此位置变为插入的entry，且将插入entry的next节点变为原来的entry
        //2、如果此位置不存在entry则直接插入新的entry
        addEntry(hash, key, value, i);
        return null;
    }

    # 7.2 取(get)过程

    1. 获取对象时，我们将K传给get，它调用hashCode计算hash从而得到bucket位置，并进一步调用equals()方法确定键值对。如果发生碰撞的时候，Hashmap通过链表将产生碰撞冲突的元素组织起来，在Java 8中，如果一个bucket中碰撞冲突的元素超过某个限制(默认是8)，则使用红黑树来替换链表，从而提高速度
    2. get方法的工作原理：通过对key的hashCode()进行hashing，并计算下标( n-1 & hash)，从而获得buckets的位置。如果产生碰撞，则利用key.equals()方法去链表中查找对应的节点
    //根据key获得一个entry
public V get(Object key) {
        //如果key为null，获取0号位的且key为null的值
        if (key == null)
            return getForNullKey();
        //如果不是，获取entry，在下面的方法中
        Entry<K,V> entry = getEntry(key);
        //合法性判断
        return null == entry ? null : entry.getValue();
    }
    
//获取一个key不为null的entry
final Entry<K,V> getEntry(Object key) {
        //如果table为null，则返回null
        if (size == 0) {
            return null;
        }
        //计算hash值
        int hash = (key == null) ? 0 : hash(key);
        //根据hash值获得table的下标，遍历链表，寻找key，找到则返回
        for (Entry<K,V> e = table[indexFor(hash, table.length)];
             e != null;
             e = e.next) {
            Object k;
            if (e.hash == hash &&
                ((k = e.key) == key || (key != null && key.equals(k))))
                return e;
        }
        return null;
}

    # 8.hashmap如何处理碰撞的，或者说说它的扩容？

    1. hashMap 降低哈希碰撞的方法：
      • static final int hash(Object key) //扰动函数 最终的目的是让散列分布地更加均匀
      • 扩容，将原来的数组长度变成原来的二倍
    2. hashmap 实现扩容的逻辑：hashmap在 put 的时候先判断table不为空、且新增元素的key不存在，于是进行新增一个Entry的逻辑：
    void addEntry(int hash, K key, V value, int bucketIndex) { //进行新增一个Entry的逻辑
        if ((size >= threshold) && (null != table[bucketIndex])) {
            resize(2 * table.length);
            hash = (null != key) ? hash(key) : 0;
            bucketIndex = indexFor(hash, table.length);
        }
        createEntry(hash, key, value, bucketIndex);
    }

    由源代码可知，在新增元素时，会先判断： A.当前的entry数量是否大于或者等于阈值（loadfactory*capacity) B.判断当前table的位置是否存在entry 经上两个条件联合判定，才会进行数组的扩容工作，最后扩容完成才会去创建新的entry，而扩容的方法即为：resize（），以下为源代码

    void resize(int newCapacity) {
        //拿到原table对象
        Entry[] oldTable = table;
        //计算原table的桶长度
        int oldCapacity = oldTable.length;
        //先判定，当前容量是否已经是最大容量了（2的30次方）
        if (oldCapacity == MAXIMUM_CAPACITY) {
            //假如达到了，将阈值设为int的最大值2的31次方减1，返回
            threshold = Integer.MAX_VALUE;
            return;
        }
        //创建新的table对象
        Entry[] newTable = new Entry[newCapacity];
        //将旧的table放入新的table中
        transfer(newTable, initHashSeedAsNeeded(newCapacity));
        //赋值新table
        table = newTable;
        //计算新的阈值
        threshold = (int)Math.min(newCapacity * loadFactor, MAXIMUM_CAPACITY + 1);
    }
//具体的扩容过程
void transfer(Entry[] newTable, boolean rehash) {
        int newCapacity = newTable.length;
        //遍历原table，重新散列
        for (Entry<K,V> e : table) {
            while(null != e) {
                Entry<K,V> next = e.next;
                if (rehash) {
                    e.hash = null == e.key ? 0 : hash(e.key);
                }
                int i = indexFor(e.hash, newCapacity);
                e.next = newTable[i];
                newTable[i] = e;
                e = next;
            }
        }
    }

    # 9.HashMap底层数组长度扩容为什么是2的幂次方数——为了使hash均匀分组

    当数组长度为2的n次幂的时候，不同的key算得得index相同的几率较小，那么数据在数组上分布就比较均匀，也就是说碰撞的几率小，相对的，查询的时候就不用遍历某个位置上的链表，这样查询效率也就较高了

    # 10.HashMap 怎样解决冲突？

    HashMap中处理冲突的方法实际就是链地址法，内部数据结构是数组+单链表

    # 11.当两个对象的hashcode相同会发生什么？

    两个对象的Hashcode相同，所以它们的bucket位置相同，会发生“碰撞”。HashMap使用链表存储对象，这个Entry(包含有键值对的Map.Entry对象)会存储在链表中

    # 12.如果两个键的hashcode相同，如何获取值对象？

    通过对key的hashCode()进行hashing，并计算下标( n-1 & hash)，从而获得buckets的位置。此时两个键的hashcode相同会产生碰撞，则利用key.equals()方法去链表或树中去查找对应的节点

    # 13.针对 HashMap 中某个 Entry 链太长，查找的时间复杂度可能达到 O(n)，怎么优化？

    将链表转为红黑树，实现 O(logn) 时间复杂度内查找。JDK1.8 已经实现了。

    # 14.如果HashMap的大小超过了负载因子(load factor)定义的容量，怎么办？

    （1）扩容：容量扩充为原来的两倍（2 * table.length）
    （2）移动：对每个节点重新计算哈希值，重新计算每个元素在数组中的位置，将原来的元素移动到新的哈希表中(这是一个非常消耗性能的操作）

    正因为扩容后的移动操作非常消耗性能，所以如果我们已经预知hashmap中元素的个数，那么预设元素的个数能够有效的提高hashmap的性能 比如说，我们有1000个元素new HashMap(1000), 但是理论上来讲new HashMap(1024)更合适，不过扩容是2的幂次方数，即使是1000，hashmap也自动会将其设置为1024。 但是new HashMap(1024)还不是更合适的，因为0.75*1000 < 1000, 也就是说为了让0.75 * size > 1000, 我们必须这样new HashMap(2048)才最合适，既考虑了&的问题，也避免了resize的问题

    # 15.为什么String, Interger这样的类适合作为键？

    因为String对象是不可变的，而且已经重写了equals()和hashCode()方法了

    （1）不可变性是必要的，因为为了要计算hashCode()，就要防止键值改变，如果键值在放入时和获取时返回不同的hashcode的话，那么就不能从HashMap中找到你想要的对象。不可变性还有其他的优点如线程安全
    （2）因为获取对象的时候要用到equals()和hashCode()方法，那么键对象正确的重写这两个方法是非常重要的。如果两个不相等的对象返回不同的hashcode的话，那么碰撞的几率就会小些，这样就能提高HashMap的性能

    # 16.HashMap与HashTable区别

    1. HashMap继承于AbstractMap，而Hashtable继承于Dictionary
    public class HashMap<K,V> extends AbstractMap<K,V>
    implements Map<K,V>, Cloneable, Serializable {
    1

    1. 线程安全不同。Hashtable的几乎所有函数都是同步的，即它是线程安全的，支持多线程。而HashMap的函数则是非同步的，它不是线程安全的。若要在多线程中使用HashMap，需要我们额外的进行同步处理
    2. null值。HashMap的key、value都可以为null。Hashtable的key、value都不可以为null
    3. 迭代器(Iterator)。HashMap的迭代器(Iterator)是fail-fast迭代器，而Hashtable的enumerator迭代器不是fail-fast的。所以当有其它线程改变了HashMap的结构（增加或者移除元素），将会抛出ConcurrentModificationException
    4. 容量的初始值和增加方式都不一样：HashMap默认的容量大小是16；增加容量时，每次将容量变为“原始容量x2”。Hashtable默认的容量大小是11；增加容量时，每次将容量变为“原始容量x2 + 1”
    5. 添加key-value时的hash值算法不同：HashMap添加元素时，是使用自定义的哈希算法。Hashtable没有自定义哈希算法，而直接采用的key的hashCode()
    6. 速度。由于Hashtable是线程安全的也是synchronized，所以在单线程环境下它比HashMap要慢。如果你不需要同步，只需要单一线程，那么使用HashMap性能要好过Hashtable

    # 17.如何让HashMap变得线程安全（同步）？

    使用 synchronize 可以使 HashMap 进行同步 (参考: https://blog.csdn.net/u012512634/article/details/72735183)

    Map m = Collections.synchronizeMap(hashMap);

    # 18.HashMap，LinkedHashMap，TreeMap 有什么区别？

    1. HashMap就是上面讲的HashMap
    2. LinkedHashMap 保存了记录的插入顺序，在用 Iterator 遍历时，先取到的记录肯定是先插入的；遍历比 HashMap 慢
    3. TreeMap 实现 SortMap 接口，能够把它保存的记录根据键排序（默认按键值升序排序，也可以指定排序的比较器）

    # 19.HashMap，LinkedHashMap，TreeMap的使用场景？

    1. 一般情况下，使用最多的是 HashMap，完成在 Map 中插入、删除和定位元素等基本功能
    2. LinkedHashMap：在需要输出的顺序和输入的顺序相同的情况下
    3. TreeMap：在需要按自然顺序或自定义顺序遍历键的情况下

    # 20.Java 中的另一个线程安全的与 HashMap 极其类似的类是什么？同样是线程安全，它与 HashTable 在线程同步上有什么不同？

    1. ConcurrentHashMap 类（是 Java并发包 java.util.concurrent 中提供的一个线程安全且高效的 HashMap 实现）
    2. HashTable 是使用 synchronize 关键字加锁的原理（就是对对象加锁）
    3. ConcurrentHashMap在 JDK 1.7 中采用 分段锁的方式；JDK 1.8 中直接采用了CAS（无锁算法）+ synchronized

    # 21.HashMap & ConcurrentHashMap 的区别？

    除了加锁，原理上无太大区别。在用法上，HashMap 的键值对允许有null，但是ConCurrentHashMap 都不允许。

    # 22.为什么 ConcurrentHashMap 比 HashTable 效率要高？

    1. HashTable 使用一把锁（锁住整个链表结构）处理并发问题，多个线程竞争一把锁，容易阻塞
    2. ConcurrentHashMap 在 JDK1.7中使用分段锁，相当于把一个 HashMap 分成多个段，每段分配一把锁，这样支持多线程访问
    3. 锁粒度：基于 Segment，包含多个 HashEntry
    4. JDK 1.8 中使用 CAS + synchronized + Node + 红黑树。锁粒度：Node（首结点）（实现 Map.Entry<K,V>）。锁粒度降低了
